Beyond the Books - Online Bookstore

CS 5244: Web Application Development

Aug 2024 - Dec 2024

A data-centric online bookstore that includes session handling, database management, and user history, designed with reusability, scalabilty, and consistency in mind.

The use of React with TypeScript was the foundation for the majority of the project. Utilizing different components for various sections and pages of the application allowed for reusability and consistency throughout the project. For example, the books were displayed by different categories, but the display of books remained consistent for each category.

REST APIs and the Data Access Object (DAO) pattern were used to create dynamically-generated web content. This was especially crucial when generating the cart and checkout pages. Using a universal cart and bookstore list, it remained consistently accessible to all the subpages in the DOM tree. The cart kept track of each book and its corresponding information, as well as the quantity of each book. Once the user decided to check out, they would enter valid information before proceeding. If the information was invalid, it would output an error message on the interface to indicate the specific error to the user. There was also additional validation on the backend to ensure that if information got changed when sending the order detail to the backend, it would catch the errors there as well.

Overall, this application allowed for dynamically-generated content while also considering design principles of reusability, consistency, scalability, and more. This project was a great experience as I was able to get hands-on practice with a full stack project, implementing everything from the frontend to the backend of the project.

BlossomDB

CS 4604: Introduction to Database Management Systems

Aug 2024 - Dec 2024

Implemented a UI which allowed a user to perform create, read, update, and delete requests to the SQL database, limiting access to different tables and database requests depending on the users role.

The project focused on creating a database structure that worked best with the flow and implementation of the tables. My group and I implemented a relational database, normalizing all the tables to allow for easy create, read, update, and delete operations.

To achieve this, we used Express.js and Node.js to connect to the MySQL database. By utilizing SQL statements, we were able to make requests to the database. Since the database was dynamically generated using Axios calls on the frontend, once the backend request was valid, users could see the changes in the database reflected in the UI. For user login restrictions, we used content-type headers to keep track of user permissions based on their roles. When each user accessed a subtable, it used that information to display only the CRUD operations they were allowed to perform.

Overall, this project focused on creating a database that would allow for easy readability as well as integration into a UI. I enjoyed learning how to perform CRUD operations on the database and using Axios to make the API calls.
